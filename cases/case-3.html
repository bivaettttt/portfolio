Модель данных для каталога

Кейс про проектирование данных: сущности/связи, бизнес-правила, ограничения и SQL-проверки модели.

Цель

Спроектировать модель данных каталога товаров: категории, товары, SKU/варианты, атрибуты, цены и остатки.

Эффект

Структура, которая предотвращает “кривые” данные, поддерживает расширение атрибутов и позволяет строить отчеты через SQL.

Моя роль

  • Определил ключевые сущности и связи (ERD)
  • Зафиксировал правила целостности и бизнес-ограничения
  • Подготовил примеры SQL-запросов для проверки модели и отчетов
  • Продумал расширяемость (атрибуты без изменения схемы)

Контекст

Каталог используется витриной и внутренними системами. Важно корректно хранить варианты (SKU), атрибуты (цвет/размер), цены (история), остатки по складам и принадлежность к категориям.

Сущности

  • Category
    иерархия категорий, parentId
  • Product
    карточка товара (название, описание, бренд)
  • SKU
    вариант товара (цвет/размер), barcode
  • Attribute
    справочник атрибутов (color, size…)
  • SKUAttribute
    значения атрибутов для SKU
  • Price
    цена по SKU (периоды действия)
  • Stock
    остатки по складам

Правила и ограничения

Целостность

• SKU обязательно связан с Product.
• Barcode уникален.
• Нельзя иметь 2 активные цены, которые пересекаются по времени.
• Остаток не может быть отрицательным.

Бизнес-правила

• Категории вложенные (дерево).
• Product может быть в нескольких категориях (M:N).
• Атрибуты расширяемые (без изменения схемы БД).

ERD (описание)

Category(id, parentId, name) — дерево категорий.
Product(id, name, brand, description).
ProductCategory(productId, categoryId) — связь M:N.
SKU(id, productId, barcode, isActive).
Attribute(id, code, name, dataType).
SKUAttribute(skuId, attributeId, value).
Price(id, skuId, amount, currency, validFrom, validTo).
Warehouse(id, name).
Stock(skuId, warehouseId, quantity, updatedAt).

Примеры SQL (проверки модели)

1) Товары в категории (включая подкатегории)

Пример через рекурсивный CTE (PostgreSQL) — собрать поддерево и вывести товары. 

WITH RECURSIVE cat AS (
  SELECT id
  FROM category
  WHERE id = :categoryId
  UNION ALL
  SELECT c.id
  FROM category c
  JOIN cat ON c.parent_id = cat.id
)
SELECT p.id, p.name
FROM product p
JOIN product_category pc ON pc.product_id = p.id
JOIN cat ON cat.id = pc.category_id
ORDER BY p.name;
2) Текущая цена по SKU
SELECT sku_id, amount, currency
FROM price
WHERE sku_id = :skuId
  AND now() >= valid_from
  AND (valid_to IS NULL OR now() < valid_to)
ORDER BY valid_from DESC
LIMIT 1;
3) Топ-10 SKU с минимальными остатками
SELECT s.sku_id, SUM(s.quantity) AS total_qty
FROM stock s
GROUP BY s.sku_id
ORDER BY total_qty ASC
LIMIT 10;

Критерии приемки (пример)

  • AC-01
    Нельзя создать SKU без productId
  • AC-02
    Barcode уникален, дубликаты запрещены
  • AC-03
    Нет пересекающихся активных цен
  • AC-04
    Stock.quantity не допускает отрицательных значений
